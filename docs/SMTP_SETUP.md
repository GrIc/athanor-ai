# SMTP Setup — Gmail App Password

## Overview

Athanor uses Gmail SMTP to send parental alerts (real-time) and weekly digests.
This guide explains how to configure it securely using a Gmail App Password.

## What is an App Password?

An App Password is a 16-character password generated by Google that allows
third-party apps to access your Gmail account **without** using your main password.
It is scoped to a specific app and can be revoked independently.

## Prerequisites

- A Gmail account (the one that will **send** alerts)
- 2-Step Verification enabled on that Google account (required for App Passwords)

## Step-by-Step Setup

### 1. Enable 2-Step Verification

If not already enabled:
1. Go to https://myaccount.google.com/security
2. Under "How you sign in to Google", click **2-Step Verification**
3. Follow the setup wizard

### 2. Generate an App Password

1. Go to https://myaccount.google.com/apppasswords
2. Under "App passwords", click **Create** (or "Select app" > "Other")
3. Enter a name: `Athanor Monitoring`
4. Click **Create**
5. Google displays a 16-character password like: `abcd efgh ijkl mnop`
6. **Copy it without spaces**: `abcdefghijklmnop`

### 3. Configure Terraform

#### For CI/CD (GitHub Actions)

```bash
# Set secrets in your GitHub repository
gh secret set TF_VAR_SMTP_USER --body "your-email@gmail.com"
gh secret set TF_VAR_SMTP_PASSWORD --body "abcdefghijklmnop"
gh secret set TF_VAR_MONITORED_USER_EMAILS --body "child1@gmail.com,child2@gmail.com"
gh secret set TF_VAR_PARENT_ALERT_EMAIL --body "parent@gmail.com"
```

#### For local deployment

```bash
cp infra/envs/.env.prod.example infra/envs/.env.prod
# Edit .env.prod:
export TF_VAR_smtp_user="your-email@gmail.com"
export TF_VAR_smtp_password="abcdefghijklmnop"
export TF_VAR_monitored_user_emails="child1@gmail.com"
export TF_VAR_parent_alert_email="parent@gmail.com"

source infra/envs/.env.prod
```

### 4. Deploy

```bash
terraform -chdir=infra apply -var-file=envs/prod.tfvars
```

## Troubleshooting

### "Authentication failed" error

- Verify the App Password is correct (16 chars, no spaces)
- Ensure 2-Step Verification is enabled
- Check that the Gmail address in `TF_VAR_SMTP_USER` matches the account that generated the App Password

### "Connection refused" or timeout

- Cloud Run Jobs need outbound internet access (default: allowed)
- Gmail SMTP uses port 587 (STARTTLS) — ensure no firewall rule blocks it

### Emails go to spam

- Add your Gmail as a trusted sender in the parent's email client
- Consider using a custom domain with SPF/DKIM records for better deliverability

## Quotas

| Limit | Value |
|-------|-------|
| Emails per day (Gmail free) | ~500 |
| Emails per second | ~1 (rate-limited by cooldown) |

For parental monitoring, 500 emails/day is far more than needed.
If you ever exceed this, consider switching to SendGrid or Mailgun.

## Security Notes

- The App Password is stored in **GCP Secret Manager**, never in plaintext
- It is only accessible by the Cloud Run Job's service account
- Revoke the App Password anytime from https://myaccount.google.com/apppasswords
